게임 프로그래밍 수행평가

1번

WWW는 ( World Wide Web )의 약어.

2번

HTML은 ( HyperText Markup Language )의 약어.

3번

HTTP는 WWW 상에서 정보를 주고받을 수 있으며, HTTP는 ( 클라이언트 )와 서버 사이에 이루어지는 ( 요청 / 응답 ) 을 제어하는 ( 프로토콜 )이다.

4번

HTTPS는

  1. HyperText Transfer Protocol over Secure Socket Layer의 약자이다.
  2. HTTP의 보안이 강화된 버전이다.
  3. 초기 전자 상거래에서 데이터 보안 목적으로 사용되었다.
  4. 일반 데이터에도 변조 방지 목적으로 사용처가 늘어 나고 있다.

5번

S3 ( Server Side Script ) 에 대한 설명이다.

  • WWW에서 사용되는 스크립트 언어 중 서버 사이드에서 실행되는 스크립트 언어이다.
  • ASP, JSP, PHP, Node.JS, CGI등 다양햔 종류의 프로그래밍 언어가 해당된다.
  • 몇몇 프로그래밍 언어는 WWW 상이 아닌 다른 용도로도 많이 사용된다.

6번

WAS ( 웹애플리케이션서버 ) 에 대한 설명이다.

  • 인터넷 상에서 HTTP를 통해 사용자 컴퓨터나 장치에 애플리케이션을 수행해 주는 미들웨어이다.
  • 웹 애플리케이션 서버는 동적 서버 콘텐츠를 수행하는 것으로 일반적인 웹 서버와 구별이 된다.
  • 주로 데이터베이스 서버와 같이 수행된다.

7번

이미지 서버/CDN 서버에 대한 설명이다.

웹 서비스의 초창기에는 이미지 서버와 ( S3 서버 )를 따로 두지 않았으나, 데이터 처리량이 많아지면서 전문적인 기능에 따라 서버를 따로 두어 ( 처리속도 )를 향상시키는 경우가 많다. 데이터 변화가 작은 이미지나 텍스트 데이터 또는 스트리밍 데이터를 포함하기도 하며, 간혹 ( 사용자 인증 )기능이 들어가 있기도 한다.

8번

쿠키는

  1. 하이퍼텍스트의 기록서의 일종이다.
  2. 사이트의 서버를 통해 인터넷 사용자의 컴퓨터에 설치되는 작은 기록 정보 파일이다.
  3. HTTP 쿠키, 웹 쿠키, 브라우저 쿠키라고도 한다.
  4. 이 기록 파일에 담긴 정보는 인터넷 사용자가 같은 웹사이트를 방문할 때마다 읽히지만 새로운 정보로 바뀐다.
  5. 오늘날 많은 서버 및 웹사이트들이 브라우저 신속성을 위해 거의 사용한다.

9번

세션에 대한 설명이다.

세션은 ( 사용자 정보 )가 서버 메모리에 저장하여 사용되기 때문에 해커에 의한 위변조가 쿠키보다 어렵다고 한다. 이 때문에 보안이 중요한 서비스에 종종 사용되기도 하지만 ( 서버 메모리 )를 과다하게 사용하거나 다른 서버로 접속할 때는 ( 웹브라우저 )에 사용자 정보가 없어 재인증을 거쳐야 하는 불편이 있기도 하여 대용량 웹사이트에서는 쿠키와 병행하여 사용하기도 한다.

10번

JAVA에 대한 설명이다.

썬 마이크로시스템즈의 제임스 고슬링과 다른 연구원들이 개발한 ( 객체지향적 ) 프로그래밍 언어이다. 자바의 개발자들은 유닉스 기반의 배경을 가지고 있었기 때문에 문법적인 특성은 C++의 조상인 C언어와 비슷하다.

11번

XML에 대한 설명이다.

XML은 W3C에서 개발된 ( 마크업 언어 ) 로써 다른 많은 종류의 데이터를 기술하고 통신하는데 사용할 수 있다. XML은 주로 다른 종류의 시스템, 특히 인터넷에 연결된 시스템끼리 데이터를 쉽게 주고받을 수 있게 하여 HTML의 한계를 극복할 목적으로 만들어졌다.

12번

HTTP/HTTPS 서버 응답코드에 대한 설명이다.

  • 클라이언트와 서버 간에 http/https 프로토콜로 통신하는 경우 서버의 상태나 요청에 대한 응답 코드를 받게 되어 있다.
  • 보안이 강화되고 있어 모든 기능들은 프로토콜에 따라 전부 승인되지 않을 수 있다.

13번

HTTP/HTTPS 서버 응답 코드

2xx    :    성공
3xx    :    리다이렉션 완료
4xx    :    요청 오류
5xx    :    서버 오류

14번

JavaScript에 대한 설명이다.

( 객체 기반 )의 프로그래밍 언어이다. 이 언어는 웹 브라우저 내에서 주로 사용하며, 다른 응용프로그램의 ( 내장 객체 ) 에도 접근 할 수 있는 기능을 가지고 있다. 또한 Node.JS와 같은 ( 런타임 ) 환경과 같이 서버 사이드 네트워크 프로그래밍에도 사용되고 있다.

15번

PHP에 대한 설명이다.

1995년 처음 펄로 제작된 홈페이지를 제작 툴로 만들어져 발전된 ( 스크립트 ) 방식의 언어이다. 펄과 C언어에 영향을 받아 문법구조는 C언어와 비슷하다. php눈 자체 함수를 따로 ( DDL )이나 SO로 만들어 사용하고자 하는 함수만 ( PHP 코어 )에 넣어 사용할 수 있다.

16번

JSON(제이슨)은 속성정보와 데이터 값으로 이루어진 데이터 오브젝트를 전달하기 위해 인간이 읽을 수 있는 텍스트를 사용하는 ( 개방형 표준 포맷 ) 이다.

17번

Node.JS에 대한 설명이다.

  • 확장성 있는 네트워크 애플리케이션 개발에 사용되는 소프트웨어 플랫폼이다.
  • 작성 언어로는 자바스크립트를 활용하며 웹브라우저 제작사마다 자바스크립트 동작 특성 때문에 스크립트가 동작하지 않거나 원하지 않는 결과가 나올 수 있어, 어느 특정 웹브라우저의 자바 스크립트에 구애 받지 않고 자바스크립트를 개발 할 수 있는 이점이 있다.

18번

HTTP server에 대한 설명이다.

HTTP 프로토콜을 해석하고 해석된 데이터를 토대로 ( 브라우저 )에게 정보를 전달하는 서버를 통칭하는 것으로 서버는 정적 서버와 동적 서버로 나뉘게 된다.

19번

MySQL에 대한 설명이다.

오픈 소스 기반의 관계형 데이터베이스 관리 시스템이다. 데이터를 저장하고 추출하고, 변경하는 등 사용법이 간단하고 교육 문서가 많아 세계에서 가장 많이 쓰이고 있는 데이터베이스이기도 하다.

20번

서버 사이드 스크립트 언어에 대한 설명이다.

서버에서 동작하는 스크립트 언어이다. 브라우저가 HTTP 서버에 정보를 요청할 때 서버는 단순히 저장되어 있는 정보를 주는 것 이외에 능동적으로 웹브라우저의 상태에 따라서 서로 다른 정보를 가공하여 보여줘야 할 필요성이 대두되었다. 이에 개발된 언어가 ASP(.NEP), PHP, Java, Node.JS 등 다양한 스크립트 언어가 개발 되었다.
